Synchronized关键字是如何生效的？

分类：并发编程， 作者： 徐梦旗， 发布于： 2024-04-22 20:00， 字数：419，预计阅读：1分钟
#源码

1. 背景

1.1. 为什么需要并发编程？

利用多线程的优势，可以提高程序运行的性能。

1.2. 多线程带来哪些问题和风险？

多线程带来了以下问题：

  • CPU缓存带来的可见性问题。
  • 指令重排序带来的有序性问题。
  • 线程切换带来的原子性问题。

多线程带来了以下风险：

  • 死锁。
  • 活锁。
  • 饥饿。

1.3. 如何解决多线程带来的问题？

  • Java内存模型。
  • Happen-Before原则。
  • 加锁。

2. 锁

2.1. 锁的分类

锁有以下分类：

  • 按照：公平锁和非公平锁
  • ：乐观锁和悲观锁
  • ：共享锁（读锁）和排查锁（写锁）
  • ：可重入锁和非可重入锁
  • ：可中断锁和非可中断锁
  • ：自旋锁和非自旋锁

2.2. 锁优化

  • 锁分离
  • 降低锁的粒度
  • 缩小锁的范围
  • 锁粗化
  • 无锁

3. synchronized解析

3.1. synchronized是如何使用的？

synchronized关键字是Java编程语言提供的内置锁。

  • 加在代码块上
  • 加在静态方法
  • 加载非静态方法

3.2. synchronized的底层原理是？

对象头组成

  • Markword
  • 类指针
  • 数组指针
  • 实例数据
  • 对齐填充

4. synchronized锁升级的过程是怎样的？

4.1. 无锁

创建对象时是无锁

4.2. 偏向锁

当只有一个线程竞争锁时通过CAS切换线程ID，无锁算法，自旋

4.3. 轻量级锁

当多个线程竞争锁时，CAS切换线程指针

4.4. 重量级锁

当有线程长时间获取不到锁时，创建Monitor锁
