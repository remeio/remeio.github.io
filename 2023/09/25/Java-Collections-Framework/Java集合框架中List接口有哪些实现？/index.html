Java集合框架中List接口有哪些实现？

分类：Java-Collections-Framework， 作者： 徐梦旗， 发布于： 2023-09-25 20:00， 字数：2.3k，预计阅读：9分钟
#源码

1. List接口详解

1.1. List接口有哪些操作？

List列表是一个有序的Collection，有时也被叫做序列（sequence）。List可能包含重复的元素。除了从Collection接口继承的操作外，List接口还包括以下操作：

  • 位置访问（Positional access）
    通过数字下标操作列表中的元素。包括的方法有get，set，add，addAll，remove。

  • 搜索（Search）
    在列表中搜索指定的对象，并返回其对应的数字下标。搜索方法包括indexOf和lastIndexOf。

  • 迭代（Iteration）
    继承Iterator接口的语义。使用listIterator方法可以返回一个迭代器。

  • 范围视图（Range-view）
    sublist方法可对指定范围的列表进行操作。

Java平台提供了两种常规用途的List接口的实现，分别是ArrayList和LinkedList。ArrayList在一般情况下有更好性能，LinkedList在特定场景下有更好的性能。

2. ArrayList详解

2.1. 有了数组为什么还需要ArrayList？

  1. 在声明数组时需要指定数组的大小，而ArrayList则不需要；
  2. ArrayList提供了一系列封装好的操作，比直接操作数组更方便。

2.2. ArrayList底层是如何实现的？

  • 从继承关系上看，ArrayList实现了List接口和RandomAccess接口；
  • 从底层数据结构上看，ArrayList基于动态数组，当容量不足时会自动扩容；
  • 从存储元素上看，ArrayList允许存储null元素以及存储重复的元素；
  • 从线程安全上看，ArrayList是线程不安全的，采用fast-fail机制。

2.3. ArrayList是如何扩容的？

  1. 在添加新元素前，如果最小容量大于当前数组的大小，则触发扩容，否者直接添加新元素（如果是无参构造器声明的ArrayList，首次扩容时，最小容量为10）；
  2. 把当前数组的大小的1.5倍作为新容量（如果新容量仍小于最小容量，则取最小容量作为新容量）；
  3. 调用Arrays#copyOf方法进行数组扩容后，添加新元素。
public boolean add(E e) {
	// 1. 添加新元素前，如果有必要则进行扩容
	ensureCapacityInternal(size + 1);  // Increments modCount!!
	elementData[size++] = e;
	return true;
}


private static int calculateCapacity(Object[] elementData, int minCapacity) {
	// 2. 如果是无参构造器声明的ArrayList，首次扩容时，最小容量为10
	if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
		return Math.max(DEFAULT_CAPACITY, minCapacity);
	}
	return minCapacity;
}

private void ensureCapacityInternal(int minCapacity) {
	ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));
}

private void ensureExplicitCapacity(int minCapacity) {
	modCount++;

	// 3. 最小容量大于当前数组的大小才进行扩容
	// overflow-conscious code
	if (minCapacity - elementData.length > 0)
		grow(minCapacity);
}


private void grow(int minCapacity) {
	// overflow-conscious code
	int oldCapacity = elementData.length;
	// 4. 当前数组数组大小的1.5倍作为新容量
	int newCapacity = oldCapacity + (oldCapacity >> 1);
	// 5. 如果新容量仍不够，取最小容量作为新容量
	if (newCapacity - minCapacity < 0)
		newCapacity = minCapacity;
	if (newCapacity - MAX_ARRAY_SIZE > 0)
		// 在最小容量不大于MAX_ARRAY_SIZE时，限制新容量不大于MAX_ARRAY_SIZE，避免虚拟机无法创建数组
		newCapacity = hugeCapacity(minCapacity);
	// minCapacity is usually close to size, so this is a win:
	// 进行数组扩容 
	elementData = Arrays.copyOf(elementData, newCapacity);
}

private static int hugeCapacity(int minCapacity) {
	if (minCapacity < 0) // overflow
		throw new OutOfMemoryError();
	return (minCapacity > MAX_ARRAY_SIZE) ?
		Integer.MAX_VALUE :
		MAX_ARRAY_SIZE;
}

2.4. ArrayList中操作的时间复杂度？

指定下标访问元素-ArrayList#get

  • 通过指定下标访问数组中的元素，是随机访问，时间复杂度为O(1)；
  • 平均时间复杂度为O(n)。
public E get(int index) {
	rangeCheck(index);
	// 下标访问数组
	return elementData(index);
}

添加新元素-ArrayList#add(E)

  • 当需要扩容时，由于需要复制数组，时间复杂度为O(n)；
  • 当不需要扩容时，时间复杂度为O(1)；
  • 平均时间复杂度为O(n)。
public boolean add(E e) {
	// 需要扩容时，会发生数组复制
	ensureCapacityInternal(size + 1);  // Increments modCount!!
	elementData[size++] = e;
	return true;
}

删除指定下标的元素-ArrayList#remove(int)

  • 当删除的是最后一个元素时，时间复杂度为O(1)；
  • 当删除的不是最后一个元素时，由于需要移动数组，时间复杂度为O(n);
  • 平均时间复杂度为O(n)。
public E remove(int index) {
	rangeCheck(index);

	modCount++;
	E oldValue = elementData(index);
	// 有且仅有删除最后一个元素时（index为size-1），不满足以下条件
	int numMoved = size - index - 1;
	if (numMoved > 0)
		// 数组移动
		System.arraycopy(elementData, index+1, elementData, index,
						 numMoved);
	elementData[--size] = null; // clear to let GC do its work

	return oldValue;
}

2.5. 什么是fast-fail机制？

fast-fail机制是指当迭代器被创建后，如果修改了列表（不包括迭代器本身的remove和add操作），迭代器会抛出ConcurrentModificationException异常。因此当并发修改时，迭代器会快速失败来避免后续的并发风险，来提示开发者避免在多线程下使用线程不安全的列表。

final void checkForComodification() {
	// 列表都会记录当前的修改次数modCount，然后会同迭代器内部记录的expectedModCount进行比较，如果不同则代表有别的线程对列表进行了修改操作
	if (modCount != expectedModCount)
		throw new ConcurrentModificationException();
}

3. LinkedList详解

3.1. LinkedList是如何实现的？

  • 从继承关系上看，LinkedList实现了List接口和Deque接口；
  • 从底层数据结构上看，LinkedList基于双向链表，无需扩容；
  • 从存储元素上看，LinkedList允许存储null元素以及存储重复的元素；
  • 从线程安全上看，LinkedList是线程不安全的，采用fast-fail机制。

3.2. LinkedList中操作的时间复杂度？

指定下标访问元素-LinkedList#get

  • 当要访问的元素在头尾节点时，时间复杂度为O(1)；
  • 当要访问的元素不在头尾节点时，由于需要遍历链表，时间复杂度为O(n)；
  • 平均时间复杂度为O(n)。
public E get(int index) {
	checkElementIndex(index);
	// 遍历链表
	return node(index).item;
}

Node<E> node(int index) {
	// assert isElementIndex(index);
	// 遍历一半的链表
	if (index < (size >> 1)) {
		Node<E> x = first;
		for (int i = 0; i < index; i++)
			x = x.next;
		return x;
	} else {
		Node<E> x = last;
		for (int i = size - 1; i > index; i--)
			x = x.prev;
		return x;
	}
}

在指定下标添加新元素-LinkedList#add(int, E)

  • 当添加的新元素在头尾节点时，时间复杂度为O(1)；
  • 当添加的新元素不在头尾节点时，由于需要遍历链表找到插入位置，时间复杂度为O(n)；
  • 平均时间复杂度为O(n)。
public void add(int index, E element) {
	checkPositionIndex(index);

	// 尾结点添加
	if (index == size)
		linkLast(element);
	// 非尾结点添加元素，需要遍历链表找到插入位置
	else
		linkBefore(element, node(index));
}

删除指定下标的元素-LinkedList#remove(int)

  • 当删除的元素在头节点时，时间复杂度为O(1)；
  • 当删除的元素不在头节点时，由于需要遍历链表找到删除位置，时间复杂度为O(n);
  • 平均时间复杂度为O(n)。
public E remove(int index) {
	checkElementIndex(index);
	return unlink(node(index));
}

3.3. ArrayList和LinkedList有什么区别？

官方建议在选择使用LinkedList时，应该同时测试ArrayList和LinkedList的性能，来验证是否真的有必要选用LinkedList，通常来说ArrayList拥有更好的性能。

  • 底层数据结构不同，ArrayList基于数组，而LinkedList基于链表。
  • 消耗内存大小不同，相比于ArrayList，LinkedList需要存储链表的节点，会消耗更多的内存空间。
  • 操作的时间复杂度不同，ArrayList是常量级访问，线性级修改，LinkedList是线性级访问，常量级修改。
  • 实现接口不同， LinkedList实现了Deque接口，可以当做双向队列使用，这个是ArrayList做不到的。

4. Vector详解

4.1. Vector底层是如何实现的？

  • 从继承关系上看，Vector实现了List接口和RandomAccess接口；
  • 从底层数据结构上看，Vector基于动态数组，当容量不足时会自动扩容；
  • 从存储元素上看，Vector允许存储null元素以及存储重复的元素；
  • 从线程安全上看，Vector是线程安全的，采用synchronized修饰了所有方法。

4.2. ArrayList和Vector有什么区别？

Vector可以理解为线程安全的ArrayList（线程安全的List的实现推荐使用CopyOnWriteArrayList）。

  • 线程安全性不同，ArrayList是线程不安全的，Vector采用了synchronized关键字修饰了所有方法，是线程安全的；
  • 扩容大小不同，ArrayList扩容为原数组大小的1.5倍，而Vector扩容为原数组的2倍；
  • 性能不同，相比于ArrayList，Vector中的方法需要加锁，性能要比ArrayList差。

4.3. Stack底层是如何实现的？

Stack继承了Vector，是线程安全的，可以当做堆栈使用（推荐使用ArrayDeque代替Stack）。
