1. Map接口详解

1.1. Map接口有哪些操作？

Map是指将键映射到对应的值。一个Map不能包含重复的键：每个键最多只能映射一个值。Map接口包括以下操作：

  • 基础操作（Basic operations）
    如put，get，remove，containsKey，containsValue，size以及empty方法。

  • 批量操作（Bulk operations）
    如putAll，clear方法。

  • 集合视图（Collection views）
    如keySet，entrySet，values方法。

2. HashMap详解

2.1. HashMap底层是怎么实现的？

  • 从底层数据结构上看，HashMap使用了数组，链表以及红黑树；
  • 从存储元素上看，HashMap允许存储null键和null值，是无序的；
  • 从线程安全上看，HashMap是线程不安全的，采用fast-fail机制。

2.2. HashMap是如何解决哈希冲突的？

哈希冲突是指两个对象经过哈希函数计算得到的哈希值相同。解决哈希冲突的方法有：

  • 开放定址法：冲突时寻找新的空闲的内存地址。
  • 再哈希法：再次计算哈希值直到不发生哈希冲突。
  • 链表法：利用链表维护哈希冲突的节点。
  • 建立公共溢出区：额外开辟一块内存区域来维护冲突的节点。

HashMap使用链表法解决哈希冲突，哈希值相同的键所对应的节点由一条链表维护。

在一定的条件下（最小树化容量为64，树化阈值为8），HashMap会将链表转为红黑树，原因是当链表中的节点越来越多时，查询的时间复杂度会由O(1)退化为O(n)；而使用红黑树时，查询的时间复杂度为O(logn)，比链表的查询效率要高，代价是新增和修改操作需要维护红黑树的结构，故在一定条件下，红黑树也会转为链表（链化阈值为6）。

2.3. HashMap是如何添加键值对的？

  1. 首先通过哈希函数计算当前键的哈希值，并通过位运算获取当前键在数组中的下标；如果该下标对应的节点为空，则创建当前键值对的节点即可。否者代表发生了哈希冲突。

  2. 发生哈希冲突后，首先判断当前下标对应的节点的哈希值和键同当前键是否相同，如果是则修改键对应的值即可。否者需要链表或红黑树解决哈希冲突。

  3. 首先判断当前下标对应的节点的类型是否是树节点，如果是则代表是红黑树，在红黑树中创建当前键值对的节点即可。否者需要通过链表解决哈希冲突。

  4. 遍历当前下标对应的节点所在的链表，在链表尾部添加当前键值对的节点，如果链表的长度大于8，且数组的大小大于64时，链表会被转为红黑树。

  5. 最后，如果键值对的个数大于扩容的阈值，即数组的大小乘以负载因子，会进行扩容。

public V put(K key, V value) {
	return putVal(hash(key), key, value, false, true);
}

static final int hash(Object key) {
	int h;
	return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
	Node<K,V>[] tab; Node<K,V> p; int n, i;
	if ((tab = table) == null || (n = tab.length) == 0)
		n = (tab = resize()).length;
	if ((p = tab[i = (n - 1) & hash]) == null)
		tab[i] = newNode(hash, key, value, null);
	else {
		Node<K,V> e; K k;
		if (p.hash == hash &&
			((k = p.key) == key || (key != null && key.equals(k))))
			e = p;
		else if (p instanceof TreeNode)
			e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
		else {
			for (int binCount = 0; ; ++binCount) {
				if ((e = p.next) == null) {
					p.next = newNode(hash, key, value, null);
					if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
						treeifyBin(tab, hash);
					break;
				}
				if (e.hash == hash &&
					((k = e.key) == key || (key != null && key.equals(k))))
					break;
				p = e;
			}
		}
		if (e != null) { // existing mapping for key
			V oldValue = e.value;
			if (!onlyIfAbsent || oldValue == null)
				e.value = value;
			afterNodeAccess(e);
			return oldValue;
		}
	}
	++modCount;
	if (++size > threshold)
		resize();
	afterNodeInsertion(evict);
	return null;
}

如果不进行扩容，当元素较多时，会频繁发生哈希冲突，导致大量的元素存放在链表或红黑树中，导致查询效率较差。扩容会

2.5. HashMap是如何根据键找到值的？

首先通过哈希函数计算当前键的哈希值，并通过位运算获取当前键在数组中的下标；如果该下标对应的节点不为空，则遍历链表或红黑树查找。

判断当前下标对应的节点的类型是否是树节点，如果是则代表是红黑树，在红黑树中查找当前键对应的值。否者需要遍历链表。

public V get(Object key) {
	Node<K,V> e;
	return (e = getNode(hash(key), key)) == null ? null : e.value;
}

final Node<K,V> getNode(int hash, Object key) {
	Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
	if ((tab = table) != null && (n = tab.length) > 0 &&
		(first = tab[(n - 1) & hash]) != null) {
		if (first.hash == hash && // always check first node
			((k = first.key) == key || (key != null && key.equals(k))))
			return first;
		if ((e = first.next) != null) {
			if (first instanceof TreeNode)
				return ((TreeNode<K,V>)first).getTreeNode(hash, key);
			do {
				if (e.hash == hash &&
					((k = e.key) == key || (key != null && key.equals(k))))
					return e;
			} while ((e = e.next) != null);
		}
	}
	return null;
}
