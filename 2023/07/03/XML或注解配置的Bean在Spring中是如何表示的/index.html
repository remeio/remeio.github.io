XML或注解配置的Bean在Spring中是如何表示的？

分类：SpringFramework， 发布时间： 2023-07-03 15:26:00
#源码

背景

为什么要搞清楚Bean在Spring中是如何表示的，主要有下面两个方面的原因：

  1. 对于使用者来说，在使用XML或注解配置Bean的时候，我们需要知道有哪些可配置的信息，才能进行配置；
  2. 对于开发者来说，在IOC容器处理Bean的流程中，会依赖Bean在Spring中的表示信息，是阅读源码的前置知识。

如果让你设计，你会如何设计？

我们使用Spring是为了让Spring帮我们管理依赖，Bean也就是依赖的别称。那创建一个依赖需要哪些信息呢？不难想到以下这些信息：

  • 类的名称
  • 属性的值，如果有的话
  • 构造器参数的值，如果有的话

如果说我们还需要Spring帮我们做一些额外动作，如Bean创建或销毁时分别执行某个方法，我们可以添加以下信息：

  • 初始化方法
  • 销毁方法

如果说我们并不仅限于使用构造器或属性的方式创建一个依赖，而是通过工厂的方式创建依赖，我们可以添加以下信息：

  • factory-bean，用于指定工厂
  • factory-method，用于指定工厂中创建Bean的方法
  • …

不难发现，对于开发者来说，当我们需要越多的功能，对Bean的表示也就越复杂。而对于使用者来说，也就有了更多可配置也就是可自定义功能的地方。

Spring为了更好更全面地满足用户的需求，不得不增加了许多不常用的功能，这样的好处是复杂度转移给了Spring的开发者，而使用者只需关注自己的业务，根据业务做相应的配置即可，这也是一个成熟的企业级框架的设计思想。

Spring是如何设计Bean的表示的？

同上面我们的设计类似，不过我们的设计的基础上，Spring还维护了如是否延迟初始化、作用域、所需依赖等信息。体现在源码上便是BeanDefinition接口，如下类图。

从继承关系上看，BeanDefinition继承了以下接口：

  • BeanMetadataElement接口，该接口的getSource方法会返回Bean的元数据来源于哪里，也就是这个Bean的信息最初是在哪里设置的，如在XML中或配置类中。

  • AttributeAccessor接口，用于Spring容器记录一些内部的属性供后续处理过程中使用。

从实现上来看，BeanDefinition有以下子类：

  • 通用的抽象实现AbstractBeanDefinition
  • 通过XML加载的GenericBeanDefinition
  • 通过@Component等注解标注的ScannedGenericBeanDefinition或AnnotatedGenericBeanDefinition
  • 通过@Configuration注解标注的ConfigurationClassBeanDefinition
  • 通过@Bean注解标注的AnnotatedGenericBeanDefinition
  • 容器处理过程中使用的RootBeanDefinition

为什么要拆分为这么多的BeanDefinition？

起初Spring只有基于XML配置的Bean，使用的是RootBeanDefinition的继承体系；后来为了支持基于注解配置Bean的方式，在Spring2.5引入了GenericBeanDefinition的继承体系。

不同的Bean来源所携带的信息并不完全相同，如使用@Component配置的Bean相比于使用XML配置的Bean就多了注解携带的一些信息，所以才需要使用不同的BeanDefinition的子类来满足不同的来源。

需要注意的是最终BeanDefinition的类型取决于该BeanDefinition的处理方式，同样是@Component标注的Bean，如果使用组件扫描的方式则会得到ScannedGenericBeanDefinition，如果编程式注册到应用上下文的方式则会得到AnnotatedGenericBeanDefinition，详见下文实战。

源码剖析-如何表示构造器参数？

从设计上来看，构造器参数无非就参数的下标，及参数的值两部分。

可以通过维护一个Map来维护多个构造器参数，键就代表参数的下标，值就代表参数的值；另外还需要额外添加一个List来维护泛型参数。

ConstructorArgumentValues#ConstructorArgumentValues(ConstructorArgumentValues) 使用了原型模式，提供了同类型构造器以深拷贝对象。

源码剖析-如何表示属性值？

从设计上来看，属性值无非就属性的名称，及属性的值两部分。

可以通过PropertyValue类来代表一个属性值，其包含了属性的名称，属性的值，转换后的值，以及是否已被转换等字段。

为了方便维护多个属性值，我们可以通过MutablePropertyValues来持有一个PropertyValue的列表。为了方便对属性值的遍历，我们可以实现了PropertyValues接口，本质也是一个PropertyValue的迭代器。

MutablePropertyValues使用了迭代器模式，方便对多个属性值进行遍历；并维护了多个属性值为其提供统一的访问操作，也是一种组合模式。

源码剖析-如何实现通用的BeanDefinition？

BeanDefinition的通用抽象实现是AbstractBeanDefinition，也是所有BeanDefinition实现类的父类，维护了Bean定义所包含的信息，提供了一系列对Bean定义中的信息的设置及获取操作。

源码剖析-如何实现通用的基于注解的BeanDefinition

基于注解的BeanDefinition的通用实现是AnnotatedGenericBeanDefinition，其维护了注解的元数据，也就是标注在该类上的所有注解的相关信息。

AnnotatedTypeMetadata代表了所标注注解的元数据，ClassMetadata代表了类的元数据；MethodMetadata代表了方法的元数据，而AnnotationMetadata是一个组合接口，代表了所标注注解的元数据，类的元数据和方法的元数据。

我们可以使用Builder模式通过BeanDefinitionBuilder来构建各种类型的BeanDefinition。如GenericBeanDefinition，RootBeanDefinition，和ChildBeanDefinition。

实战-不同的配置Bean的方式会构建哪种类型的BeanDefinition？

通过XML加载的GenericBeanDefinition

1.首先我们先创建一个简单的实体类，它包含name和age两个属性。

@Slf4j
@Data
public class TestBean {

    private String name;

    private int age;
}

2.我们通过XML的方式声明一个Bean。

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="testBean" class="com.remeio.upsnippet.spring.beandefinition.TestBean">
        <property name="name" value="foo"/>
        <property name="age" value="1"/>
    </bean>
</beans>

3.加载该XML，并打印出testBean的BeanDefinition的信息

public static void testGenericBeanDefinition() {
	ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("beans.xml");
	context.refresh();

	final BeanDefinition beanDefinitionOfTestBean = context.getBeanFactory().getBeanDefinition("testBean");
	final TestBean testBean = context.getBean(TestBean.class);
	log.info("bean definition: {}， bean: {}", beanDefinitionOfTestBean, testBean);
}
// bean definition: Generic bean: class [com.remeio.upsnippet.spring.beandefinition.TestBean]; scope=; abstract=false; lazyInit=false; autowireMode=0; dependencyCheck=0; autowireCandidate=true; primary=false; factoryBeanName=null; factoryMethodName=null; initMethodName=null; destroyMethodName=null; defined in class path resource [beans.xml]， bean: TestBean(name=foo, age=1)

4.从运行结果中我们可以看到它所对应的BeanDefinition的类型是GenericBeanDefinition。

它的propertyValues字段也和我们在XML中配置的值相同。
